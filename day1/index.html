Part 2

Types of Machine Learning

The "type" of ML answers one question: What kind of data did the model learn from? There are four types. Learn the intuition for each, not just the label.

2.1 Supervised Learning

The intuition: Imagine teaching a child what a cat is by showing them 1,000 photos labeled "cat" and 1,000 photos labeled "not cat." The labels are the supervision. The child (model) learns to map photo → label.

How it works mechanically:

  • Input data has a known correct answer called a label
  • The model learns the mapping input → output
  • After training, give it a new unseen input and it predicts the output

Two flavors of supervised learning:

FlavorOutput typeExample
ClassificationA category (discrete)Spam or not spam
RegressionA number (continuous)Predicted house price: $427,500

Real-world examples:

  • Loan approval: features (age, salary, credit score) → label (approved / rejected)
  • Tumor diagnosis: image → label (malignant / benign)
  • Demand forecast: historical sales → label (next week's sales number)
🔑 Exam clue words: "labeled data", "historical data with known outcomes", "predict category", "predict value", "training examples include the answer"

2.2 Unsupervised Learning

The intuition: Same child, but now you dump a pile of photos in front of them with no labels and say: "Sort these into groups however you want." The child might separate by color, by animal type, by background. They find structure that was already hidden in the data — without anyone telling them what's correct.

How it works mechanically:

  • Input data has no labels at all
  • The model finds hidden structure: clusters, patterns, outliers
  • There's no "right answer" to compare against — the model discovers structure on its own

Real-world examples:

  • Customer segmentation: group 10 million customers into "buying personas" you never defined upfront
  • Topic discovery: find recurring themes in a million news articles
  • Anomaly detection: find the 0.01% of credit card transactions that "look weird" compared to everything else
🔑 Exam clue words: "unlabeled data", "discover patterns", "group similar items", "segment customers", "no predefined categories"

2.3 Reinforcement Learning (RL)

The intuition: Think of training a dog. The dog tries something → you give a treat (reward) or say "no" (penalty). Over many trials, the dog learns which actions lead to treats. There is no labeled dataset — there is a goal and feedback after each action.

How it works mechanically:

  • An agent takes actions in an environment
  • Each action returns a reward (positive) or penalty (negative)
  • The agent's goal: learn a policy (a strategy) that maximizes total reward over time
  • Learning happens entirely via trial and error — no human-labeled data needed

Real-world examples:

  • Game-playing AI: AlphaGo, chess engines — reward = winning the game
  • Robotics: a robot learning to walk — reward = staying upright and moving forward
  • Autonomous driving simulation
  • Dynamic pricing: raise/lower price → observe sales → adjust strategy
  • Ad bidding strategies
🔑 Exam clue words: "agent", "environment", "reward", "policy", "trial and error", "learn through interaction"

2.4 Semi-supervised Learning

The intuition: You have 1,000 medical X-rays carefully labeled by a radiologist (very expensive and slow!) and 100,000 unlabeled X-rays sitting in a database. Throwing away 100,000 examples is wasteful. Semi-supervised learning uses both the labeled and unlabeled data together.

How it works mechanically:

  • Small amount of labeled data + large amount of unlabeled data
  • Model uses the unlabeled data to learn general structure, and the labeled data to anchor meaning
  • Best of both worlds when human labeling is expensive or slow

Real-world examples:

  • Medical imaging: labeling requires a specialist doctor (expensive)
  • Speech recognition: transcription is slow and costly
  • Web page classification: millions of pages exist, but very few are labeled
🔑 Exam clue words: "limited labeled data", "large unlabeled dataset", "reduce labeling cost", "labeling is expensive"

Part 3

ML Problem Types

The "problem type" answers a different question: What shape is the output?

This is separate from the ML type. A supervised problem can be classification or regression — those are different problem types. Think of ML type as how the model learned, and problem type as what it produces.

3.1 Classification — predict a category

The output is discrete — a finite set of choices. You're asking the model to put each input into a bucket.

InputOutput
Email textSpam / Not spam
Bank transactionFraud / Legitimate
X-ray imageDisease / No disease / Inconclusive
  • Binary classification: exactly 2 classes (spam vs not spam, fraud vs legitimate)
  • Multi-class classification: 3 or more classes (cat / dog / bird)
  • Multi-label classification: multiple labels per input (an image can be "beach" AND "sunset" AND "people" simultaneously)
🔑 Exam clue words: "which class?", "category", "yes/no", "type of object", "label", "predict whether"

3.2 Regression — predict a number

The output is continuous — a number anywhere on a scale. There's no fixed set of buckets.

InputOutput
House details (sq ft, location, age)Price: $427,500
Last 30 days of salesRevenue forecast: $1.2M
Weather sensor dataEnergy demand: 450 MWh
💡 Key distinction: "Will this customer churn?" = Classification (yes/no). "How many days until they churn?" = Regression (a number).
🔑 Exam clue words: "predict price", "forecast demand", "estimate value", "numeric output", "how much", "how many"

3.3 Clustering — group similar things (unsupervised)

No labels exist. The model creates its own groups based on similarity. You don't tell it how many groups or what they mean — it figures that out.

  • Customer segmentation: find 5 natural buyer personas from 10M customers
  • Document grouping: organize 1M articles into themes
  • Genome analysis: group similar gene expression profiles
🔑 Exam clue words: "group similar", "segment", "no labels", "natural groupings", "discover clusters"

3.4 Anomaly Detection — find the weird stuff

Find rare events that don't match the normal pattern. The challenge is that you usually have very few anomaly examples — so pure supervised learning struggles here. Often treated as unsupervised: "learn what normal looks like, then flag anything that doesn't fit."

  • Unusual login: different country, unusual time, new device
  • Fraudulent transaction: purchase pattern doesn't match user history
  • Machine failure: vibration sensor showing abnormal readings
  • Network intrusion: traffic pattern looks unlike historical baseline
🔑 Exam clue words: "unusual", "outlier", "abnormal", "rare event", "deviation from normal", "strange spike"

3.5 Recommendation — suggest relevant items

Predict what a user will like based on their past behavior and the behavior of similar users. This is called collaborative filtering — "people like you also liked X."

  • Netflix: "Because you watched X, you might like Y"
  • Amazon: "Customers who bought this also bought…"
  • Spotify Discover Weekly
  • News article recommendations
🔑 Exam clue words: "personalized", "users who bought X also bought Y", "next best item", "recommend", "suggest"

Putting it all together

ProblemML TypeProblem Type
Spam filterSupervisedClassification
Predict tomorrow's stock priceSupervisedRegression
Group products by similarityUnsupervisedClustering
Find rare network attacksUnsupervised (usually)Anomaly Detection
"You may also like…"SpecializedRecommendation
Part 4

The ML Pipeline — 7 Stages

This is the lifecycle of every ML system — from raw data to a working, monitored product. Memorize the order. The exam tests whether you can identify which stage a described problem belongs to.

1. Data Collection ↓ 2. Preprocessing (cleaning) ↓ 3. Feature Engineering ↓ 4. Training ↓ 5. Evaluation ↓ 6. Deployment ↓ 7. Monitoring ↓ (back to 4 when retraining needed)
1
Stage 1 — Data Collection
Gather raw data from wherever it lives: databases, application logs, images, audio recordings, IoT sensors, clickstream data, or public datasets. The quality and quantity of data here determines the ceiling of model performance.

⚠️ Garbage in, garbage out. ML is not magic. If your data is biased, incomplete, or wrong, your model will be too. This is not fixable by clever algorithms later. The exam loves this — if a question says "model is biased," the answer is almost always fix the data, not tune the model.
2
Stage 2 — Preprocessing (Cleaning)
Raw data is messy. You clean it so the model can actually use it. This is often the most time-consuming step in practice.

TaskWhat it does
Remove duplicatesThe same row twice misleads training — the model thinks it's more important than it is
Handle missing valuesEither fill with a substitute (imputation: mean, median, mode) or drop the row/column
Normalize / standardizeScale numeric values so they're on comparable ranges. E.g., height in cm (170) vs salary in $ (70,000) — wildly different scales break many algorithms
Convert formatsImage → tensor, text → tokens, dates → timestamps
Remove noiseDrop irrelevant or corrupted rows
Encode categories"Country = India" can't be fed directly to most models → convert to one-hot encoding or embeddings
3
Stage 3 — Feature Engineering
Transform raw fields into useful features — the inputs that actually help the model learn the pattern you care about. A model can't always extract every useful pattern from raw data on its own. Smart features make patterns obvious.

Examples:
  • Raw: date_of_birth = 1995-03-12 → Feature: age = 30
  • Raw: transaction_timestamp → Features: hour_of_day, day_of_week, is_weekend
  • Raw: address → Features: city, pincode, is_metro
  • Raw text → TF-IDF vectors or word embeddings
4
Stage 4 — Training
The model learns by adjusting its internal numbers (parameters) to minimize the difference between its predictions and the correct answers.

ComponentMeaning
AlgorithmThe learning method — e.g., XGBoost, neural network, k-means
Training dataThe examples used to teach (with labels, for supervised)
ParametersInternal numbers the model learns — weights, biases
Loss functionMeasures how wrong the predictions are (the "score" to minimize)
OptimizerThe mechanism that adjusts parameters to reduce loss — e.g., gradient descent
HyperparametersSettings you choose before training: learning rate, batch size, number of layers
5
Stage 5 — Evaluation
Test the trained model on data it has never seen before. This is how you know if it actually learned a general pattern or just memorized the training examples. The specific metrics you use depend on the problem type — we cover all of these in Part 7.
6
Stage 6 — Deployment
Put the model into production so users and applications can actually use it. The right deployment mode depends on how fast you need an answer and how big the inputs are.

Deployment TypeWhen to useExample
Real-time inferenceNeed an answer in under 1 secondFraud check during checkout
Batch inferenceScore millions of records, no urgencyMonthly churn scoring overnight
Async inferenceLarge input, processing takes minutesVideo analysis, large documents
Edge deploymentModel runs on the device itselfPhone camera, IoT sensor
7
Stage 7 — Monitoring
After deployment, watch for problems. The world changes, and your model — trained on old data — may stop performing well.

IssueWhat it means
Data driftInput data has changed — e.g., a new product line the model was never trained on
Model drift / concept driftThe relationship between input and output has changed — e.g., COVID changed shopping patterns overnight
BiasModel is treating groups unfairly — e.g., approving loans for one demographic more than another
LatencyPredictions are getting slower over time
CostThe endpoint is too expensive to run
ErrorsFailed inference requests are increasing
When monitoring detects problems, you retrain the model with fresh data. That loop back to Stage 4 is the full MLOps cycle.
Part 5

Overfitting vs Underfitting

This is the #1 ML concept on the exam. It will appear in multiple questions, sometimes disguised. Master the intuitions, the signs, and especially the fixes.

5.1 Overfitting

The intuition: A student who memorizes the answer key rather than understanding the subject. They ace every practice test (training data) but fail the real exam (new data) because they never actually learned the underlying concepts.

What the model does: It learns the noise in the training data — the random quirks specific to those examples — instead of the general pattern. It fits the training data so perfectly that it can't generalize to anything new.

How to spot it:

  • Training accuracy: 99%
  • Validation/Test accuracy: 65%
  • The gap is large — this is the defining sign of overfitting

Common causes:

  • Model is too complex (too many parameters for the amount of data you have)
  • Not enough training data
  • Trained for too many epochs (iterations)
  • Too many features, some of which are just noise

Fixes — memorize all of these, the exam picks from them:

FixHow it helps
More training dataMuch harder to memorize a larger, more diverse dataset
Regularization (L1, L2)Adds a penalty to the loss function for overly complex models, forcing simpler solutions
Reduce model complexitySmaller network, fewer layers, fewer features — less capacity to memorize
Dropout (neural nets)Randomly "turns off" neurons during training so the model can't rely too much on any single path
Early stoppingStop training when validation accuracy stops improving — don't let the model keep memorizing
Cross-validationBetter estimate of true performance; helps detect overfitting earlier
Data augmentationCreate variations of existing data so the model sees more diversity — harder to overfit

5.2 Underfitting

The intuition: A student who studied only the chapter titles. They perform badly on practice tests and badly on the real exam, because they never learned enough to begin with.

What the model does: The model is too simple to capture the real pattern in the data. It makes large, systematic errors on both training and test data.

How to spot it:

  • Training accuracy: 60%
  • Validation/Test accuracy: 58%
  • Both are low — the gap doesn't matter, the absolute numbers are just bad

Common causes:

  • Model is too simple for the complexity of the data
  • Not enough or poorly chosen features
  • Training stopped too early
  • Wrong algorithm for the problem

Fixes:

  • Use a more complex model (more layers, more parameters)
  • Add more or better features (feature engineering)
  • Train longer (more epochs)
  • Reduce regularization — if regularization is too aggressive it can cause underfitting
🚨 Exam trap: A very common wrong answer is "add regularization" when the scenario describes underfitting. Regularization is the fix for overfitting. For underfitting, you reduce regularization and increase complexity.

Part 6

Bias vs Variance Tradeoff

This is a deeper lens on overfitting and underfitting. Once you understand it, the two concepts click together permanently.

6.1 Bias — error from oversimplified assumptions

The model is too rigid. It makes strong assumptions about the shape of the data that are simply wrong, so no matter how much data you give it, it can't improve beyond a ceiling.

Example: Imagine the true relationship between house size and price is a curve. You try to fit a straight line to this data. No matter how many houses you train on, a straight line cannot bend — it will always be wrong in a systematic way. That systematic wrongness is bias.

High bias = underfitting. The model is too simple to represent reality.

6.2 Variance — error from over-sensitivity to the training set

The model is too flexible. It bends itself to fit every single training point — including random noise — so tiny changes to the training data produce wildly different models.

Example: A super-flexible squiggly curve that passes through every training point exactly. Retrain with slightly different data and the squiggle looks completely different. That instability is variance.

High variance = overfitting. The model is too complex and memorizes noise.

6.3 The Tradeoff

You cannot easily have zero bias AND zero variance simultaneously. Reducing one tends to increase the other. The art of ML is finding the sweet spot.

SituationBiasVarianceOutcome
UnderfittingHigh ↑Low ↓Model too simple — wrong everywhere
OverfittingLow ↓High ↑Model too complex — right on training, wrong on new data
Sweet spot ✅Low ↓Low ↓Generalizes well — the goal
💡 Mnemonic: Bias = Bad assumptions (too simple). Variance = Volatile (too sensitive to the specific training set).

Evaluation Metrics

This is where many candidates lose points. The exam doesn't just ask you to define metrics — it gives you a scenario and asks which metric is appropriate. Master the "when to use which" logic.

7.1 The Confusion Matrix — the foundation of all classification metrics

For binary classification, every prediction falls into one of exactly four boxes. Every metric below is computed from these four numbers. Learn this grid cold.

Predicted POSITIVE
Predicted NEGATIVE
Actually
POSITIVE
✅ True Positive (TP)
Correctly flagged positive

NEGATIVE

✅ True Negative (TN)

Top-left and bottom-right = correct predictions. The off-diagonal = errors.

7.2 Accuracy

Accuracy = (TP + TN) / (TP + TN + FP + FN)

Plain English: Of all predictions made, what fraction were correct?

Use when: Classes are roughly balanced — close to 50/50 distribution.

⚠️ The accuracy trap — guaranteed exam question: If 99% of transactions are legitimate and 1% are fraud, a model that always predicts "legitimate" achieves 99% accuracy but catches zero fraud. It is completely useless. Never use accuracy alone on imbalanced data.

7.3 Precision

Precision = TP / (TP + FP)

Plain English: Of everything the model flagged as positive, what fraction actually was positive? How precise are your positive predictions?

Use when: False positives are expensive or harmful — you want to be sure before you call something positive.

Classic example — Spam filter: If the model marks a legitimate email as spam (FP), the user misses something important — maybe a job offer. High precision means: "When I say spam, I'm confident it really is spam."

7.4 Recall (also called Sensitivity)

Recall = TP / (TP + FN)

Plain English: Of all the actual positive cases that exist, what fraction did the model catch? How many real cases did it miss?

Use when: False negatives are expensive or dangerous — you cannot afford to miss a real case.

Classic example — Cancer screening: If the model misses a real cancer case (FN), the patient doesn't get treated. That's catastrophic. High recall means: "I catch nearly every real cancer case — I might raise false alarms, but I won't miss the real ones."

7.5 The Precision–Recall Tradeoff

You trade one for the other by adjusting the model's decision threshold — the probability cutoff above which it calls something "positive."

  • Lower the threshold → flag more things as positive → catch more real positives (↑ recall) but also more false alarms (↓ precision)
  • Raise the threshold → only flag the most certain cases → fewer false alarms (↑ precision) but miss some real cases (↓ recall)

Choose based on what's costlier in your domain: missing a real case, or raising a false alarm?

7.6 F1 Score

F1 = 2 × (Precision × Recall) / (Precision + Recall)

Plain English: The harmonic mean of precision and recall. A single balanced number. It punishes extreme imbalances — a model with 100% precision but 0% recall gets F1 = 0, not 50%.

Use when: Data is imbalanced and both false positives AND false negatives matter. This is your default for fraud detection, disease detection, and any rare-event classification.

💡 AWS exam pattern: "imbalanced fraud dataset, which metric?" → F1 or Recall. If missing fraud is the primary concern → Recall. If both matter → F1.

7.7 AUC-ROC

ROC curve: plots Recall (True Positive Rate) on Y-axis vs False Positive Rate on X-axis, at every possible threshold value.

AUC (Area Under the Curve): a single number from 0 to 1 summarizing the whole curve.

  • AUC = 1.0 → perfect classifier
  • AUC = 0.5 → no better than random guessing (diagonal line)
  • AUC < 0.5 → worse than random (flip your predictions)

Use when: binary classification, you want a threshold-independent measure, or you're comparing multiple models to pick the best one.

Plain English: "How well does this model separate positives from negatives overall, regardless of where I draw the decision line?"

7.8 Regression Metrics (for numeric output problems)

MSE — Mean Squared Erroravg of (predicted − actual)²Penalizes big errors much more than small ones — squaring magnifies large gaps

Part 8

Neural Network Basics

You don't need to derive the math. You need to know what each piece does and when to use which activation function — the exam tests this.

8.1 The Neuron — the building block

A single artificial neuron does one thing: takes a set of numbers as inputs, multiplies each by a learned weight, adds them up, adds a bias, then passes the result through an activation function.

output = activation_function( Σ (input × weight) + bias )
  • Inputs: the numerical features coming in (pixel values, word embeddings, sensor readings…)
  • Weights: learned numbers controlling how important each input is. A high weight = "this input matters a lot."
  • Bias: a learned offset that lets the neuron "fire" even when all inputs are zero — adds flexibility
  • Activation function: introduces non-linearity. Without it, the whole network collapses into a single linear equation no matter how many layers you have — useless for complex patterns.

8.2 Layers

A neural network is just neurons arranged in layers, where the output of one layer feeds into the input of the next.

LayerJob
Input layerReceives the raw features. One neuron per input feature. Does no computation — just passes data in.
Hidden layer(s)Learn intermediate representations — combinations of features that are useful for the final prediction. More hidden layers = "deeper" = deep learning.
Output layerProduces the final prediction. Its structure depends on the problem: one neuron for regression, one sigmoid neuron for binary classification, N softmax neurons for N-class classification.

8.3 Activation Functions — know which goes where

Activation functions add non-linearity so the network can learn curves, corners, and complex shapes — not just straight lines.

ActivationWhat it doesWhere it's used
ReLU
(Rectified Linear Unit)		If input > 0, output = input. If input ≤ 0, output = 0. Simple, fast, avoids the "vanishing gradient" problem.Default for hidden layers in almost all modern networks
SigmoidSquashes any number into the range 0–1. Interpretable as a probability.Output layer for binary classification — gives "probability of being positive"
SoftmaxTakes N values and converts them to N probabilities that sum to exactly 1.Output layer for multi-class classification — one probability per class
TanhSquashes to range −1 to +1. Zero-centered unlike sigmoid.Older networks, some sequence models (RNNs)

8.4 Backpropagation — how a network learns

The intuition: The network makes a prediction. You compare that prediction to the true answer and calculate the error. That error is then propagated backward through the network, layer by layer. Each weight gets nudged slightly in the direction that would have reduced the error. Repeat this millions of times and the network gradually gets better.

Step 1 — Forward pass: Input → Layer 1 → Layer 2 → ... → Prediction Step 2 — Calculate error: Compare Prediction to Actual Answer → Loss value Step 3 — Backward pass: Propagate error backward → calculate how much each weight contributed to the error Step 4 — Update weights: Nudge each weight slightly to reduce loss (gradient descent) Step 5 — Repeat: Do this for every training example, thousands of times

This process of iterating over the full training set is called an epoch. Training typically runs for many epochs until the loss stops improving.

Part 9

Deep Learning Model Types

Three architectural families dominate the exam. Learn what problem each solves and why — the exam frequently gives you a use case and asks you to pick the right architecture.

9.1 CNN — Convolutional Neural Network

Best for: images and video.

Why CNNs for images? A regular (fully connected) neural network would treat each pixel as completely independent. An image of 224×224 pixels = 50,176 inputs, and every neuron in the first layer would connect to all 50,176. This is computationally exploding, and it ignores the fact that nearby pixels are related — edges, textures, shapes are all local patterns.

A CNN instead uses convolutional filters — small grids (e.g., 3×3) that slide across the image and detect local patterns. Early filters detect edges and corners. Later filters combine those into shapes. Later still into objects. This is how the network learns to recognize a face — it first finds edges, then eyes/nose/mouth shapes, then the combination that means "face."

Use cases:

  • Image classification (cat vs dog, product category)
  • Object detection (find and locate all cars in this photo)
  • Facial recognition
  • Medical imaging (tumor detection in X-rays, MRI scans)
  • Defect detection in manufacturing
  • Self-driving car perception
🔑 Exam clue words: image, visual, pixels, object detection, computer vision, photo, video frames

9.2 RNN — Recurrent Neural Network

Best for: sequential and time-ordered data.

Why RNNs for sequences? Regular networks process each input independently — they have no memory. But for sequences, order matters and context from earlier in the sequence affects later parts. A sentence like "the bank approved the loan" has "bank" meaning a financial institution — but to know that, you need context from the rest of the sentence.

RNNs have a hidden state — a memory that gets updated at each step. The output at time t depends on the input at time t AND the hidden state carried over from time t-1.

Use cases:

  • Time-series forecasting (stock prices, sensor readings)
  • Speech recognition (audio frames over time)
  • Older text generation tasks
  • Sensor data streams

Key weakness: RNNs struggle with long-range dependencies. If a sentence is 50 words, by the time the RNN reaches word 50, the signal from word 1 has been diluted through 49 updates — it's largely forgotten. Transformers fixed this problem.

🔑 Exam clue words: sequence, time series, ordered data, temporal, stream, previous step

9.3 Transformers

Best for: language, generative AI, and everything text-related. This is the architecture behind every modern LLM.

Why Transformers dominate? Instead of processing tokens one at a time like RNNs, Transformers use an attention mechanism that lets every token in a sequence directly look at every other token simultaneously. When processing the word "bank," the model can immediately attend to "loan" and "approved" in the same sentence to resolve meaning — no matter how far apart they are.

This gives three massive advantages over RNNs:

  • Long-range dependencies: any word can directly attend to any other word — no more forgetting
  • Parallelism: all tokens are processed simultaneously, not one by one — training is dramatically faster
  • Scale: they scale beautifully to billions of parameters on massive datasets — GPT-4, Claude, Llama are all Transformers

Use cases:

  • Text generation (chatbots, story writing)
  • Translation (English → French)
  • Summarization
  • Question answering
  • Code generation
  • All Large Language Models (LLMs)
  • All Foundation Models
🔑 Exam clue words: LLM, generative AI, foundation model, attention, chatbot, summarization, text generation, GPT, Claude

Part 10

Training Data Concepts

10.1 Labeling

Adding the correct answer to each training example. This is required for supervised learning. Without labels, the model has nothing to learn from.

Labeling is often the most expensive and time-consuming part of building a supervised ML system — it requires human experts (doctors for medical data, lawyers for legal data, etc.).

AWS service: SageMaker Ground Truth. It coordinates human labelers (using a workforce from Amazon Mechanical Turk, private teams, or vendors) and uses ML to speed up the process — once enough labels exist, it uses active learning to auto-label confident cases and only send the ambiguous ones to humans.

10.2 Data Splits — the three sacred sets

You split your full dataset into three non-overlapping parts with very specific rules about how each is used:

SplitPurposeRuleTypical size
Training setThe model learns from this — parameters are adjusted using this dataUsed repeatedly during training70–80%
Validation setTune hyperparameters, compare different model architectures, pick the best versionUsed during development but never to update parameters directly10–15%
Test setFinal unbiased evaluation after everything is decidedUsed exactly once, at the very end. Never look at it during development.10–15%
⚠️ Cardinal rule: Never train on the test set. Never tune hyperparameters on the test set. If you look at the test set during development, your final accuracy number is a lie — the model has indirectly been tuned on it.

10.3 Class Imbalance

The problem: Fraud detection has 1% fraud and 99% legitimate. A model that always predicts "legitimate" gets 99% accuracy but is completely useless — it catches no fraud at all.

Fixes:

TechniqueWhat it does
Oversampling minority classDuplicate existing minority examples so they appear more often during training
Undersampling majority classRandomly drop some majority examples to bring the ratio closer to balanced
SMOTE (Synthetic Minority Over-sampling Technique)Generates new synthetic minority examples by interpolating between existing ones — smarter than just duplicating
Class weightsTell the model "a mistake on the minority class costs 99× more" — it learns to care more about getting those right
Use F1 / Recall instead of accuracyFix the metric you evaluate with, not just the data

10.4 Data Augmentation

Artificially expand your training set by creating modified versions of existing examples. The model sees more variety without you collecting new data. This reduces overfitting and improves generalization.

Data TypeAugmentation Techniques
ImagesRotate, crop, flip horizontally/vertically, change brightness/contrast, add Gaussian noise, zoom in/out
TextSynonym replacement, back-translation (translate to French then back to English), random word dropout, sentence shuffling
AudioTime-stretching (speed up/slow down), pitch-shifting, adding background noise, time-shifting
💡 Data augmentation is one of the seven overfitting fixes from Part 5 — it works by making the training set larger and more diverse, which is harder to memorize.
Part 11

Hyperparameters vs Parameters

This distinction appears constantly on the exam — in questions about training, tuning, and SageMaker features.

11.1 Parameters — what the model learns

Parameters are the internal numbers that the model adjusts automatically during training. You never set these manually — the optimizer (gradient descent) finds their optimal values by minimizing the loss function.

Examples:

  • Weights and biases in a neural network (there can be billions of these)
  • Learned word embeddings
  • Split thresholds in a decision tree

11.2 Hyperparameters — what you set

Hyperparameters are settings you choose before training begins. They control the learning process itself — how fast to learn, how big each batch of data is, how complex the model is. The model cannot learn these from data; you must choose them.

Examples:

  • Learning rate: how big each weight-update step is. Too high → overshoots, never converges. Too low → takes forever.
  • Batch size: how many training examples are processed before updating weights
  • Number of epochs: how many times the model sees the full training set
  • Number of layers / neurons per layer: the model's architectural complexity
  • Dropout rate: what fraction of neurons to randomly disable during each training step
  • Regularization strength (λ): how heavily to penalize model complexity
  • Number of trees in a random forest
  • Max tree depth

11.3 Hyperparameter Tuning

Finding the best hyperparameter values is called hyperparameter tuning (or hyperparameter optimization, HPO). You try different combinations and evaluate each on the validation set. Common strategies:

  • Grid search: try every combination in a defined grid — exhaustive but slow
  • Random search: randomly sample combinations — often finds good values faster than grid search
  • Bayesian optimization: intelligently choose next values to try based on results so far — the most efficient method
💡 AWS: SageMaker Automatic Model Tuning (AMT) — also called HPO — searches the hyperparameter space for you automatically using Bayesian optimization or random search. You define the ranges; SageMaker runs the experiments and returns the best configuration.
TypeSet byExample
ParameterModel (learned during training)Neural network weights
HyperparameterYou (chosen before training)Learning rate, batch size, number of layers
Part 12

Inference vs Training

These are the two distinct phases of every ML system's life. The exam tests whether you understand the differences and which AWS deployment mode fits each real-time scenario.

12.1 Training — the model learns

During training, the model processes the training dataset, computes its errors, and updates its weights through backpropagation. This is computationally intensive and happens before the model is ever used in production.

FactorTraining
Compute neededVery high — typically requires GPUs (often many of them)
TimeLong — hours to weeks depending on dataset and model size
CostHigh — large instances running for long periods
Data usedThe entire training dataset, processed repeatedly
OutputA trained model — a file of learned weights
FrequencyPeriodic — done once, then the model is retrained later when needed

12.2 Inference — the model predicts

During inference, the trained model takes a new, never-before-seen input and produces a prediction. No learning happens — weights are frozen. This is what happens when your product is live and users are sending requests.

FactorInference
Compute neededLower per request — but scales with request volume
TimeShould be fast — users are waiting
CostDepends on traffic volume and instance type
Data usedOne input at a time (or a small batch)
OutputA prediction — a category, a number, generated text…
FrequencyContinuous — every time a user or system sends a request

There are four ways to deploy a model in SageMaker. The exam gives a scenario and expects you to pick the right one. Memorize all four.

Part 13

MLOps Concepts

MLOps = DevOps applied to Machine Learning. It's the practice of building, deploying, and maintaining ML systems reliably, repeatably, and at scale.

13.1 Model Versioning

Every time you train a new model — with fresh data, a new architecture, or different hyperparameters — you create a new version. You must track versions carefully.

Why versioning matters:

  • Rollback: if the new model breaks production, instantly revert to the previous version
  • Compare: which version — v2 or v3 — performs better on the latest data?
  • Audit: what changed and when? Required in regulated industries (banking, healthcare)
  • Reproduce: can you recreate exactly the model that was in production on a specific date?
  • Compliance: GDPR, banking regulations, FDA processes all require model lineage tracking

13.2 CI/CD/CT for ML

StageWhat it automates
CI — Continuous IntegrationAutomatically test your code, data pipelines, and training scripts every time someone pushes a change. Catches bugs before they reach production.
CD — Continuous DeliveryAutomatically deploy new model versions safely — using canary deployments (5% of traffic to new model, monitor, then gradually increase) or blue-green deployments (run both versions, switch traffic).
CT — Continuous TrainingAutomatically retrain the model when a trigger fires — e.g., every Sunday, when accuracy drops below 90%, or when new labeled data arrives.

13.3 Monitoring in Production

After deployment, the world changes and your model gradually becomes less accurate. Monitor for multiple problem types:

What to monitorWhy it matters
Accuracy / business metricIs the model still producing correct predictions? Still generating business value?
LatencyAre predictions getting slower? Indicates a resource or complexity problem.
ErrorsAre inference requests failing? At what rate?
Data driftHas the distribution of input data shifted from training? E.g., a new product category the model never saw.
Model drift / concept driftHas the relationship between inputs and outputs changed? E.g., COVID changed shopping patterns overnight — pre-COVID models became wrong.
BiasIs the model treating demographic groups unfairly? E.g., systematically denying loans for certain zip codes.

13.4 Retraining Triggers

Monitoring detects problems. Retraining fixes them. Trigger a retrain when:

  • Accuracy drops below a defined threshold
  • New labeled data becomes available
  • Data drift is detected above a threshold
  • Business rules change (new product launch, new regulation)
  • Seasonality shifts (holiday vs non-holiday patterns)
  • Bias is detected in production predictions
  • Compliance or audit requirements mandate it
💡 AWS service: SageMaker Pipelines automates this entire loop — preprocessing → training → evaluation → deployment — as a repeatable, versioned, auditable workflow.

Pre-built AWS AI Services

These are API-based services — no training needed. You send data to the API and AWS returns the AI result. Think of them as "AI as a service." For any task these services cover, use them instead of building a custom model — they're faster, cheaper, and require zero ML expertise.

17.1 Amazon Rekognition — Images & Video

What it does: Analyzes images and videos to detect and identify visual content.

  • Face detection and analysis (age range, emotions, whether wearing glasses)
  • Face comparison and search (is this face in your database?)
  • Object and scene detection (car, beach, sunset, animal…)
  • Content moderation — detect unsafe or inappropriate content (nudity, violence, drug use)
  • Celebrity recognition
  • Text in images (e.g., reading a license plate or a sign in a photo)
  • Activity detection in video (e.g., detecting a fall)
🔑 Clue words: image/video analysis, detect faces/objects, content moderation, visual recognition

17.2 Amazon Comprehend — Text Understanding (NLP)

What it does: Understands the content and meaning of text documents using Natural Language Processing.

  • Sentiment analysis: is this review positive, negative, neutral, or mixed?
  • Entity recognition: extract names of people, places, organizations, dates, dollar amounts
  • Key phrase extraction: what are the main topics discussed?
  • Language detection: what language is this text written in?
  • Topic modeling: what themes run through a collection of documents?
  • Document classification: train a custom classifier on your own categories
  • Comprehend Medical: specialized version that extracts medical entities — drug names, dosages, medical conditions, procedures
🔑 Clue words: understand text, sentiment analysis, extract entities, key phrases, NLP on documents

17.3 Amazon Textract — Extract Data from Documents

What it does: Goes beyond basic OCR. Not only reads text from scanned documents, but also understands document structure — extracts tables with their rows and columns intact, form fields with their labels and values, key-value pairs from invoices.

  • Scanned PDF text extraction
  • Table extraction from invoices, financial statements
  • Form extraction (field name + field value) from applications, contracts
  • Handwriting recognition
  • Works on low-quality, skewed, or partially obscured scans
🚨 Textract vs Rekognition trap: Rekognition sees an image as a visual scene (faces, objects). Textract treats an image as a document and extracts its structured text content. "Extract text from a scanned invoice" → Textract, not Rekognition.
🔑 Clue words: OCR, forms, tables, invoice extraction, document data extraction, structured data from scan

17.4 Amazon Transcribe — Speech → Text

What it does: Converts audio and video files into text transcriptions.

  • Real-time transcription (live captions)

17.5 Amazon Polly — Text → Speech

What it does: The opposite of Transcribe. Converts written text into natural-sounding spoken audio.

Classic use cases: accessibility features, IVR systems ("Press 1 for billing"), audiobook generation, voice assistants.

🔑 Clue words: text to speech, natural voice, generate audio from text, voice-enable an application

17.6 Amazon Translate — Language Translation

What it does: Translates text between 75+ language pairs in real-time or batch mode.

  • Real-time translation API (for live chat translation)

  • Custom terminology — specify how brand names, product names, or jargon should be translated (don't translate "SageMaker" to a French word)
  • Active Custom Translation — fine-tune quality using your own parallel text examples
🔑 Clue words: translate languages, multilingual support, localize content, cross-language

17.7 Amazon Lex — Chatbots & Voice Bots

What it does: Builds conversational interfaces — chatbots and voice bots — using the same deep learning engine that powers Amazon Alexa.

Key concepts you must know:

  • Intent: what the user is trying to do — e.g., "BookFlight", "CheckBalance", "CancelOrder"

  • Utterance: the different ways a user might phrase the same intent — "I want to fly to NYC", "Book me a flight to New York", "Get me to New York on Friday"

Use cases: customer service chatbots, voice-controlled applications, IVR phone systems, order management bots.

Part 20

Exam Traps — Read These Twice

Each of these is worth 1–2 exam points. These are the questions that separate prepared candidates from unprepared ones. Every trap below has caused many exam failures.

Trap 1
"Accuracy is the right metric" — almost never
The exam almost never wants accuracy. If the dataset is imbalanced (fraud: 1%, disease detection, rare events) → accuracy is misleading. Use F1, Recall, or AUC-ROC. Only use accuracy when classes are roughly balanced and both error types matter equally.
Trap 2
Rekognition vs Textract — visual scene vs document
Rekognition = treats an image as a visual scene — faces, objects, scenes, emotions, unsafe content. Textract = treats an image as a document — extracts text, tables, forms, key-value pairs. "Extract line items from a scanned invoice" → Textract, not Rekognition. "Detect faces at a venue entrance" → Rekognition, not Textract.
Trap 3
Transcribe vs Polly — direction matters
Transcribe = audio/speech → text (speech recognition). Polly = text → audio/speech (speech synthesis). They are exact opposites. Mnemonic: Transcribe writes down what was said. Polly reads things out loud. The exam has given both in the same question to test whether you know the direction.
Trap 4
Comprehend vs Kendra — analyze vs search
Comprehend = analyzes the content of text (what's the sentiment? what entities are mentioned? what language is this?). Kendra = searches across a library of documents to find answers to natural-language questions. "Analyze 10,000 customer reviews for sentiment" → Comprehend. "Let employees ask questions and get answers from our internal policy documents" → Kendra.
Trap 5
Forecast vs Personalize — predict numbers vs recommend items
Forecast = predict a numeric value over time (how many units will we sell next month?). Personalize = recommend specific items to specific users (what products should we show this customer?). Both involve historical data, but Forecast produces numbers and Personalize produces ranked lists of items.
Trap 6
SageMaker vs Pre-built AI Service — when to use which
SituationChoose
There's a ready API that does exactly this taskPre-built AI service
You need a custom model on your own labeled dataSageMaker AI
You want full ML lifecycle controlSageMaker AI
Team has no ML expertise, task is standardPre-built AI service or Autopilot/Canvas
Trap 7
Real-time vs Batch vs Async Inference — scenario matching
These keywords lock in the answer:
  • "low latency", "during checkout", "live response", "immediate" → Real-time Endpoint
  • "overnight", "millions of records", "no real-time requirement", "weekly batch" → Batch Transform
  • "large input (video / 500MB file)", "long processing (5 minutes)", "async OK" → Async Inference
  • "sporadic traffic", "used occasionally", "no dedicated server" → Serverless Inference
Trap 8
Underfitting fixes ≠ Overfitting fixes — don't mix them up
A very common wrong answer choice: "add regularization" when the scenario describes underfitting (both training and validation accuracy are low). Regularization is an overfitting fix. For underfitting → reduce regularization, increase model complexity, train longer, add more/better features. Never add regularization to a model that is already too simple.
Trap 9
"Model is biased" usually means the data is biased
When a question says the model is producing biased predictions, the first instinct might be to tune the model. Wrong. The exam answer is almost always: collect more representative training data, or use SageMaker Clarify to detect and mitigate bias at the data and model level. Bias in a model almost always originates in the training data — garbage in, biased garbage out.
Trap 10
JumpStart vs Bedrock — hosted in SageMaker vs managed API
SageMaker JumpStart = a catalog of pretrained and foundation models you deploy inside SageMaker onto your own infrastructure. You can fine-tune them with your data. Amazon Bedrock = a fully managed API — you call a foundation model (Claude, Llama, Titan) via API with no infrastructure at all. Use JumpStart when you need to customize/fine-tune within SageMaker. Use Bedrock when you just want to call a foundation model as a service.

Q1A bank wants to flag suspicious transactions where 0.1% are fraud. Which evaluation metric should it prioritize and why?
Click to reveal →
Recall (or F1). Missing fraud — a false negative — means a fraudulent transaction slips through, costing money. Accuracy is meaningless on 0.1% imbalanced data (a model that never flags anything gets 99.9% accuracy but catches no fraud).
Q2A retailer has 50 million customers and wants to find natural shopping segments without predefined groups. What ML type and problem type is this?
Click to reveal →
Unsupervised learning. Clustering. No labels exist — the model discovers natural groupings on its own. "Natural groups" and "no predefined categories" are the giveaway clue words.
Q3The model gets 99% training accuracy and 71% validation accuracy. What is the diagnosis and name 3 fixes?
Click to reveal →
Overfitting. Large gap between training (99%) and validation (71%) is the defining sign. Three fixes from: get more training data, add regularization (L1/L2), apply dropout, reduce model complexity, use early stopping, apply cross-validation, add data augmentation.
Q4A company wants to extract line items and totals from scanned PDF invoices. Which AWS service?
Click to reveal →
Amazon Textract. It extracts structured data — tables, forms, key-value pairs — from scanned documents. Rekognition would be wrong here — it sees the invoice as a visual image, not a document to be parsed.
Q5A team needs to monitor a deployed model for changes in the distribution of input data over time. Which SageMaker feature?
Click to reveal →
SageMaker Model Monitor. It monitors deployed endpoints and detects data drift — when the statistical properties of inputs shift from what the model was trained on. "Monitor deployed model" + "drift" = Model Monitor.
Q6A model needs to score 50 million customer records every Sunday night. There is no real-time requirement. Which SageMaker deployment type?
Click to reveal →
SageMaker Batch Transform. Bulk offline scoring, large dataset, no urgency, scheduled run → Batch Transform. Real-time Endpoint would be wrong — nobody is waiting for an instant response.
Q7A startup wants a chatbot that handles "book a flight" — it needs to collect destination, date, and number of passengers before confirming. Which AWS service?
Click to reveal →
Amazon Lex. "Book a flight" is an intent. Destination, date, and passenger count are slots. Lex is exactly designed for this — it maps utterances to intents and collects required slot values through conversation.
Q8What is the difference between a parameter and a hyperparameter? Give one concrete example of each.
Click to reveal →
Parameters are numbers the model learns automatically during training — e.g., the weights in a neural network. Hyperparameters are settings you choose before training begins — e.g., the learning rate. You tune hyperparameters; the optimizer adjusts parameters.
Q9Which deep learning architecture powers all modern LLMs (GPT, Claude, Llama)? Why does it dominate over RNNs?
Click to reveal →
Transformers. They use an attention mechanism that lets every token directly attend to every other token simultaneously — solving the long-range dependency problem that RNNs had. They also train in parallel (RNNs are sequential), enabling them to scale to billions of parameters on massive datasets.
Q10When would you choose Amazon Bedrock over SageMaker for a generative AI project?
Click to reveal →
Bedrock when you want a managed API for a foundation model (Claude, Llama, Titan) with zero infrastructure to manage — just call the API and get responses. SageMaker when you need to train a custom model from scratch, fine-tune a foundation model with your own data in a controlled environment, or need full ML lifecycle control.
